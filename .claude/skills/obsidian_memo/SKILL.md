# Obsidianメモスキル

ユーザーが「Obsidianにメモして」と言ったら、このスキルに従って操作する。

## メモ先ファイル

```
/Users/tsuyoshi/マイドライブ（yama.tsuyoshi@gmail.com）/Obsidian_Data/ノートフォルダ/260505-筋トレ記録アプリ開発-Claude Code.md
```

## 手順

1. **ファイルを読む**（Read ツールで全文取得）
2. **今日の日付の連番を確認する**
   - ファイル内に今日の日付（`# YYYY/MM/DD`）のセクションが何個あるか数える
   - 1個もなければ `# YYYY/MM/DD`（番号なし）
   - 1個あれば `# YYYY/MM/DD-2`、2個あれば `# YYYY/MM/DD-3`、以降同様
3. **このセッションでやったことを箇条書きで要約する**
   - 会話の流れから実装・修正・調査の内容を簡潔にまとめる
   - 細かい技術詳細より「何をしたか」の概要を書く
   - 既存エントリの文体・粒度に合わせる
4. **以下のフォーマットで末尾（`### 改善項目` の手前）に追記する**

```
---
# YYYY/MM/DD（または YYYY/MM/DD-N）
### 筋トレ記録アプリ開発-Claude Code
#### （セッションの内容を表す短いタイトル）
- 箇条書きメモ1
- 箇条書きメモ2
- ...
```

## 注意事項

- **既存の内容は一切編集・削除しない**
- **このファイル以外の Obsidian ファイルは触らない**
- 開始・終了時刻・作業時間はユーザーが教えてくれた場合のみ記載し、教えてもらえなければ省略する
- 追記は `### 改善項目` セクションの直前（`---` の前）に挿入する
- 作業が完了したらユーザーに「追記しました」と報告する

## Dataviewパースエラーに注意

このファイルはObsidianのDataviewプラグインが有効。**バッククォート `` ` `` で始まり `=` が続く記述**（例: `` `==text==` `` `` `=value` ``）はDataviewインラインクエリとして解釈され、パースエラーになる。

**コードとして表記したい `=` 始まりの文字列は、バッククォートを使わず `\=` でエスケープする。**

```
NG: `==text==`  →  OK: \==text==
NG: `=value`    →  OK: \=value
```
